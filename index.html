04 — classList Toggle + Dynamic Height

Accordion FAQ

Click any question to reveal the answer. Only one panel opens at a time.

The Document Object Model (DOM) is the browser's live, in-memory representation of your HTML page as a tree of objects (nodes). JavaScript can read and modify these nodes using APIs like document.querySelector(), making pages dynamic. Without the DOM, JavaScript would have no way to interact with your HTML structure.
An event listener "wires a nerve" to a DOM node. You call element.addEventListener('click', myFunction) to tell the browser: "whenever this element is clicked, run myFunction." This is the foundation of all user interactivity — clicks, keypresses, hovers, form submissions.
Every interactive element follows the Input → Process → Output pattern.

• Input: A user action fires an event (e.g., a button click).
• Process: A JavaScript function evaluates state and runs logic.
• Output: The DOM is mutated to reflect a visible change.
var is function-scoped and prone to hoisting bugs that pollute the global scope. Modern JavaScript uses const (for values that never reassign) and let (for counters/loops that do). This signals clear intent, prevents accidental reassignment, and is block-scoped — safer and more predictable.
Imperative (Vanilla JS): You write explicit, step-by-step instructions on how to find and update DOM nodes — you are telling the browser how to do something.

Declarative (React/Vue): You define what the UI should look like based on state, and the framework handles the DOM updates automatically. Vanilla JS is imperative; React is declarative.
